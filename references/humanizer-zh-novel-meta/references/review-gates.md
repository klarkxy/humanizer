# 审查闸门

生成草案给用户验收，或准备可选分享产物时，使用本文件。

## 草案验收摘要

给用户展示以下摘要。除非用户要求，不要整篇粘贴草案：

```text
作品：...
slug：humanizer-zh-...
输入校验：...章 / ...中文字符
抽样：...章，覆盖开局/推进/高潮/收束
成品规模：...条风格规则，...条去 AI 味规则，...条短摘录，约...行
覆盖审计：证据 10-16 / 规则 20-35 / 示例 3-6 / 六类规则池覆盖
摘录合规：单条≤200，总量≤2400
默认改写策略：重度作品风改写，不新增剧情事实，不迁移世界观
将写入：references/humanizer-zh-.../
是否更新根 SKILL 索引：否/待确认
```

只提供这三个结果选项：

- 批准入库
- 先修改草案
- 仅本地保留，不入库

## 公开入库标准

公开作品风格目录只包含 `SKILL.md` 和 `_meta.json`。

`SKILL.md` 必须：

- frontmatter 只包含 `name` 和 `description`。
- 目录名与 frontmatter `name` 一致。
- 没有未替换占位符。
- 单条原文摘录最多 200 个中文字符。
- 公开原文摘录总量最多 2400 个中文字符。
- 原文证据 10-16 条。
- 去 AI 味规则 20-35 条，并覆盖规则池 6 个主类。
- 完整工作示例 3-6 组。
- 包含“常见场景配方 / 改写流程 / 禁区 / 快速改写检查”。
- 推荐 500-700 行；少于 450 行通常视为过薄，超过 900 行需用户确认。
- 默认执行重度作品风改写，同时保留用户事实和世界观术语。
- 默认只输出 `【改写后】`。

`_meta.json` 必须：

- 只包含公开摘要字段。
- 不包含原文路径、本地证据路径、原文全文或规则快照路径。
- `privacy.contains_full_text` 和 `privacy.contains_source_path` 都必须是 `false`。

## 可选分享

只有用户明确要求分享或提交该风格后，才准备 issue 或 PR 文本。

共享新增风格时，预期进入版本控制的改动只有：

- 根 `SKILL.md` 的 YAML 索引条目。
- `references/humanizer-zh-<slug>/SKILL.md`。
- `references/humanizer-zh-<slug>/_meta.json`。
