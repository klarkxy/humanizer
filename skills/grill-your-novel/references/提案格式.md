# 提案格式

## 基本原则

`.grill/待确认修改.md` 是准备写入作者可读 canon 的正式 proposal 载体。每条 proposal 必须有稳定 ID，作者可以逐条确认、驳回或要求改写。

`.grill/` 文件可自动维护；作者可读 canon 文件必须经 proposal 确认后才写入。

## Proposal 格式

```markdown
## PENDING-001

目标文件：世界书.md
类型：新增 / 修改 / 删除 / 状态更新
来源：grilling 确认 / 正文后同步 / 正文反推 / 旧材料接入
状态：待确认

### 内容

- 

### 理由

- 
```

## 状态

- `待确认`：等待作者确认。
- `已确认`：作者确认，准备写入或已经写入 canon。
- `已应用`：已写入 canon。
- `已驳回`：作者拒绝，不得反复提出同一方向。
- `需改写`：作者要求修改后再确认。

## 作者确认方式

作者可以说：

- 确认全部。
- 确认 PENDING-001 和 PENDING-003。
- 驳回 PENDING-002。
- PENDING-004 改成……再给我确认。

## 驳回记录

被驳回 proposal 不要直接删除。保留在 `.grill/待确认修改.md` 的“已驳回”区，或同步摘要到 `.grill/决策记录.md`。

```markdown
## 已驳回

### PENDING-002

目标文件：人物卡/角色名.md
驳回原因：
处理结果：
```

## 正文反推标记

从正式正文中低推理得到的 canon 建议，来源写为 `正文反推`。高推理不得写入 proposal，必须先 grilling。
