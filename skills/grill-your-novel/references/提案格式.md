# 提案格式

## 基本原则

`.grill/待确认修改.md` 是要写进作者可读 canon 的正式 proposal 载体。每条 proposal 都得有稳定 ID，作者可以逐条确认、驳回、要求改写。

`.grill/` 下的文件自动维护就行；作者可读的 canon 文件必须走 proposal 确认，不能直接写。

## Proposal 格式

```markdown
## PENDING-001

目标文件：世界书.md
类型：新增 / 修改 / 删除 / 状态更新
来源：grilling 确认 / 正文后同步 / 正文反推 / 旧材料接入
状态：待确认

### 内容

- 

### 理由

- 
```

## 状态

- `待确认`：等作者确认。
- `已确认`：作者确认了，准备写或已经写进 canon。
- `已应用`：已经写进 canon。
- `已驳回`：作者拒了，别再反复提同一个方向。
- `需改写`：作者让改完再确认。

## 作者确认方式

作者可以这么回：

- 确认全部。
- 确认 PENDING-001 和 PENDING-003。
- 驳回 PENDING-002。
- PENDING-004 改成……再给我确认。

## 驳回记录

被驳回的 proposal 别直接删。留在 `.grill/待确认修改.md` 的"已驳回"区，或者把摘要同步到 `.grill/决策记录.md`。

```markdown
## 已驳回

### PENDING-002

目标文件：人物卡/角色名.md
驳回原因：
处理结果：
```

## 正文反推标记

从正式正文里低推理得出的 canon 建议，来源写 `正文反推`。高推理别写进 proposal，得先回 grilling。
