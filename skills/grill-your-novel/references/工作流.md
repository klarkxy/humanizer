# 工作流

## 目标

本 skill 服务作者本人，用中文追问式流程维护商业长篇连载网文项目。它要在多段对话之间恢复现场，持续推进当前目标，并避免未确认设定污染 canon。

## 状态文件

`.grill/` 内文件由 Agent 自动维护，不需要作者逐条确认。

- `状态.md`：当前目标、对象、阶段、门禁状态、最近上下文。
- `TODO.md`：跨对话任务队列，区分创作任务和维护任务。
- `未决问题.md`：尚未确认但暂不阻塞当前目标的问题。
- `决策记录.md`：作者已经确认、驳回或放弃的选择。
- `冲突记录.md`：材料之间的矛盾、缺口和需要追问的地方。
- `待确认修改.md`：准备写入 canon 的正式 proposal 载体。

## 推进方式

1. 先读项目入口文件和 `.grill/` 状态。
2. 明确用户当前目标，不确定时追问一个问题。
3. 检查当前目标最低必要资料。
4. 发现硬性阻塞时停止推进，进入 grilling。
5. 发现高风险但可补偿项时明确质疑，并给出修正方向。
6. 作者坚持高风险选择时，停止正面否定，转入补偿设计。
7. 不阻塞当前目标的问题，写入 `.grill/未决问题.md` 或 `.grill/TODO.md`。

## 提问格式

关键问题不要只问开放题。使用：

```markdown
问题：……

推荐：A

A. ……：叙事后果……
B. ……：叙事后果……
C. ……：叙事后果……

也可以提出其他路径。
```

每次只问一个真正阻塞推进的问题。不要一次抛出大量问卷。

## 已有材料接入

已有项目和新项目使用同一流程。读取旧正文、旧设定、旧大纲、聊天记录或人物表后：

1. 提取明确事实。
2. 标记冲突和缺口。
3. 将高推理点转成 grilling 问题。
4. 将低推理事实写入 `.grill/待确认修改.md`。
5. 作者逐条确认后再写入 canon。

## 正文后同步

每章正文完成后必须同步：

- 新人物、地点、组织、道具、规则。
- 人物目标、关系、秘密、能力、称呼和语气变化。
- 叙述口吻、节奏、场景处理、人物声音规则。
- 新伏笔、已推进伏笔、已兑现承诺、读者可感知期待。
- 时间线推进、章节状态、后续 TODO。

同步结果先进入 `.grill/待确认修改.md`，不得直接写入 canon。
