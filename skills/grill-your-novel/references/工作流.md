# 工作流

## 目标

这个 skill 给自己（作者）用，中文追问式流程，维护中文创作项目。默认服务商业长篇连载网文，也能处理短篇小说、系列短篇、散文、随笔、知乎回答、问答型观点文章和非虚构叙事。多段对话之间要能恢复现场、推进当前目标，别让没确认的设定溜进 canon。

## 状态文件

`.grill/` 里的文件由 Agent 自动维护，作者不用逐条确认。

- `状态.md`：当前目标、对象、阶段、门禁状态、最近上下文。
- `TODO.md`：跨对话任务队列，创作任务和维护任务分开。
- `未决问题.md`：还没确认但暂时不阻塞当前目标的问题。
- `决策记录.md`：作者已经确认、驳回、放弃过的选择。
- `冲突记录.md`：材料之间的矛盾、缺口、要追问的点。
- `待确认修改.md`：准备写入 canon 的正式 proposal 都在这里。

## 推进方式

1. 先读项目入口文件和 `.grill/` 状态。
2. 摸清用户当前目标和作品形态，不清楚就追问一个。
3. 看当前目标最少需要哪些资料；长篇看章节工程，短篇看单篇闭环，散文/随笔看主题、材料、声音和收束，知乎回答看问题边界、立场、论据和反驳点。
4. 碰到硬性阻塞就停下，进入 grilling。
5. 碰到高风险但能补偿的项，先质疑，给修正方向。
6. 作者坚持要高风险路线时别再正面否定，转去做补偿设计。
7. 不阻塞当前目标的问题丢进 `.grill/未决问题.md` 或 `.grill/TODO.md`。

## 提问格式

关键问题别只丢开放题。用这种结构：

```markdown
问题：……

推荐：A

A. ……：叙事后果……
B. ……：叙事后果……
C. ……：叙事后果……

也可以提出其他路径。
```

一次只问一个真正卡住推进的问题，别一股脑把问卷砸过去。

## 已有材料接入

老项目和新项目走同一套流程。读了旧正文、旧设定、旧大纲、聊天记录或人物表之后：

1. 抽出明确事实。
2. 标出冲突和缺口。
3. 高推理点转成 grilling 问题。
4. 低推理事实写到 `.grill/待确认修改.md`。
5. 作者逐条确认后再写进 canon。

## 正文后同步

每章正文写完都得同步：

- 新出现的人物、地点、组织、道具、规则。
- 人物目标、关系、秘密、能力、称呼、语气的变化。
- 叙述口吻、节奏、场景处理、人物声音规则。
- 新伏笔、推进过的伏笔、已经兑现的承诺、读者能感知到的期待。
- 时间线推进、章节状态、后续 TODO。

同步结果先到 `.grill/待确认修改.md`，不要直接写 canon。
