# 项目总览

## 题目

为什么修仙界没有 GitHub 之类的开源组织？

## 作品形态

知乎回答式短篇小说，约 10000 字。

## 核心设定

现代日常之外存在少量修仙者。中国山中有修仙门派，欧洲仍有西幻魔法世界。修仙界和凡人世界都知道编程与互联网，但修仙者不天然属于程序员圈子；年轻徒弟只是下山后听说了开源组织，并被这种共享机制震动。

## 叙事容器

一夜山中问答。徒弟下山归来，在雨夜向师父追问：凡人世界有开源，为什么修仙界没有类似组织？师父以刻薄、事故导向的方式拆解这个问题。

## 视角

第三人称。

## 人物

- 徒弟：年轻、热血、刚接触开源概念，认为公开功法可以减少散修和小门派的困境。不是程序员圈内人，不密集使用技术黑话。
- 师父：老修士，懂凡人技术，也懂西边魔法世界和魔网历史。全程阴阳怪气，老登味重，但关键处会承认门派闭源也有罪。

## 主题

修仙界没有 GitHub 式开源组织，不是因为闭源无罪，也不是因为开源天然幼稚，而是因为功法公开需要三样东西：可兼容的运行环境、可信的分发与验错机制、出了人命以后肯认错担责的共同秩序。修仙界恰好都缺。

## 论证链

1. 功法不是书，而是会进入身体、改写经脉丹田神魂的东西。
2. 每个修士的经脉、灵根、师承、灵气环境不同，功法无法像普通知识一样无差别分发。
3. 修仙界没有统一可信的来源、验印、版本、禁忌和事故记录。
4. 徒弟提出可以主动建立组织；师父用西边魔网和密斯特拉作对照，说明这种组织背后需要共同承认的权威、秩序和长期维护。
5. 真正难的不是把功法放出来，而是天下山门愿不愿意承认同一把尺。
6. 闭源也会害人。散修死在残篇里，小门派死在不完整传承里，名门也会把害人的禁忌锁进藏经阁。
7. 最冷的答案：修仙界最缺的不是功法，而是害死人以后肯认错的人。

## 魔网使用原则

魔网只作为中段对照，不喧宾夺主。明确出现魔网、密斯特拉、卡萨斯之灾，但服务于“共同秩序和公共信任层”的论证，不写成 D&D 科普文。

## 结尾方向

冷结尾。徒弟不再急着问为什么没有。师父承认闭源有罪，但指出建立这种组织的第一步不是公开功法，而是让第一个害死人的门派承认是自己的功法害死了人。
